## Headline

The current UI sits at *competent dark-SaaS*, short of an award standard — held back by a few credibility-killers on the screens that matter most: the **live exam renders as a ~75% empty void**, the **Progress dashboard reports zeros next to real activity**, the **cookie bar overlaps/blocks the footer legal links**, and a **WCAG contrast/focus cluster** undercuts a product that advertises accessibility. None require a repaint — the recommended direction (**FLIGHT DECK**) evolves the existing amber/cyan/navy cockpit tokens into a disciplined glass-cockpit avionics suite, with one ownable signature: the tri-state **Examiner Annunciator** (speaking / listening / your turn).

## Next step

Implementation (W6B.3) is **gated on your approval of the direction**. The spec proposes shipping P0 credibility fixes first (cookie bar, exam void/header, progress data diagnosis), then the token+typography sweep, then the new annunciator console behind a flag — each as a CI-green PR, paying users never broken.
